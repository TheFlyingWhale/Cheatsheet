Classes

Classes are templates for objects and a tool that we can use to quickly produce similar objects.

Constructor

Javascript calls a constructor method when we create a new instance of a class.

        
    class NewClass{
        constructor(name){
            this._name = name
        }
    }

Inheritance

Inheritance is when we create a parent class with properties and methods that we can extend to child classes.

We use the extends keyword to create a subclass.

The super keyword calls the constructor() of a parent class.

        
    class ParentClass{
        constructor(name){
            this._name = name
        }

        get name(){
            return this._name
        }
    }

    class ChildClass extends ParentClass{
        constructor(name, age){
            super(name)
            this._age = age
        }
        
        get age(){
            return this._age
        }
    }

Static methods

Static methods are called on the class, but not on instances of the class.

        
    class NewClass{
        constructor(name, age){
            this._name = name
            this._age = age
        }

        get name(){
            return this._name
        }

        get age(){
            return this._age
        }

        getInfo(){
            return `${this._name} is ${this._age} years old`
        }

        static randomName(strArr){
            return strArr[Math.floor(Math.random()* strArr.length)]
        }
    }

Modules

Modules in Node.js are reusable pieces of code that can be exported from one program and imported for use in another program.

  • module.exports

  • require()

ES6 introduced a more flexible, easier syntax to export modules:

  • default exports use export default to export JavaScript objects, functions, and primitive data types.

  • named exports use the export keyword to export data in variables.

  • named exports can be aliased with the as keyword.

  • import is a keyword that imports any object, function, or data type.

The script tag in HTML needs to have the type="module" attribute for modules to work

Node.js examples:

Export
        
    let Course = {}
    Course.name = 'Javascript Node.js'
    module.exports = Course
Import
        
    var Course = require('./moduleLocation.js')
    console.log(Course.name)

ES6 Examples:

Default export
        
    let Course = {}
    Course.name = 'Javascript ES6'
    export default Course
Default import
        
    import Course from './moduleLocation'
    console.log(Course.name)
Named exports
        
    let Course = {}
    let Student = {}
    Course.name = 'Javascript ES6'
    Student.name = 'Ole Walberg'
    export {Course, Student}

or

        
    export let Course = {}
    export let Student = {}
    Course.name = 'Javascript ES6'
    Student.name = 'Ole Walberg'
Named imports
        
    import {Course, Student} from './moduleLocation'
    console.log(Course.name)
    console.log(Student.name)
Export as
        
    let superLongNameCourse = {}
    let superLangNameStudent = {}
    ...
    export {superLongNameCourse as Course, superLangNameStudent as Student}
        
    
        
    import {Course, Student} from './moduleLocation'
    console.log(Course.name)
    console.log(Student.name)
Import as
        
    export let superLongNameCourse = {}
    export let superLongNameStudent = {}
    superLongNameCourse.name = 'Javascript ES6'
    superLangNameStudent.name = 'Ole Walberg'
        
    
        
    import {superLongNameCourse as Course, superLangNameStudent as Student} from './moduleLocation'
    console.log(Course.name)
    console.log(Student.name)
